# AGENTS.md — Wisata CI4

A tourism ticket booking system built on CodeIgniter 4.6, PHP 8.1+, MySQL.

## Quick commands

```bash
# dev server (default http://localhost:8080)
php spark serve

# run all tests
vendor/bin/phpunit

# lint/format
vendor/bin/php-cs-fixer fix

# cache
php spark cache:clear
```

## Architecture

| Layer | Location | Convention |
|---|---|---|
| Controllers | `app/Controllers/` | PascalCase, extend `BaseController` |
| Models | `app/Models/` | `M_[Name].php`, use CI4 Query Builder, not `$this->find()` |
| Views (public) | `app/Views/` | main: `landing_page.php` |
| Views (visitor) | `app/Views/visitor/` | layout: `visitor/layout.php` |
| Views (admin) | `app/Views/backend/` | Template/ dir |
| Routes | `app/Config/Routes.php` | explicit routing only, **no auto-routing** |
| Web root | `public/` | front controller: `public/index.php` |
| Assets | `public/Assets/` | CSS/JS/images |
| QR codes | `public/Assets/qrcode/` | auto-generated by `endroid/qr-code` |
| Profile uploads | `public/uploads/profile/` | |

## Database

- **DB**: MySQL, database `wisata_db` (configured in `.env`)
- **Tables**: all prefixed `tbl_` (`tbl_pengunjung`, `tbl_destinasi`, `tbl_kategori`, `tbl_transaksi`, `tbl_temp_transaksi`, `tbl_detail_transaksi`)
- **Soft deletes**: via `is_delete` column (values `'0'` / `'1'`), not CI4's built-in soft delete
- **Custom ID format**: auto-generated with prefix + zero-padded number
  - `P00001` (pengunjung), `DST001` (destinasi), `K00001` (kategori), `TR0001` (transaksi)
- **Migrations**: in `app/Database/Migrations/` (run via `php spark migrate`)
- **Seeds**: in `app/Database/Seeds/`

## Auth

- Session-based auth, not Myth/Auth or Shield
- Auth gate: `session()->get('is_login')` boolean
- Roles: `admin` and `pengunjung` (stored in `tbl_pengunjung.role`)
- Protected routes use `'auth'` filter alias → `App\Filters\AuthFilter::class`
- Route groups: `/booking/*`, `/user/*` require auth
- Admin routes (`/admin/*`) have **no filter** — admin auth checked manually inside `Admin` controller

## Model style

Models **do not rely on CI4 Model's built-in CRUD** (`find()`, `insert()`, etc.). Instead they use `$this->db->table('tbl_xxx')` directly with `getResultArray()` / `getRowArray()`. Always follow this pattern when adding model methods.

## Testing

- PHPUnit config: `phpunit.xml.dist` (bootstrap: `vendor/codeigniter4/framework/system/Test/bootstrap.php`)
- Tests in `tests/` directory
- Test DB defaults to SQLite in-memory when `ENVIRONMENT === 'testing'`
